## 1. 桌面端打包（Tauri）

桌面端基于 Tauri 2.x，前端 Vue 3 + Vite，后端 Rust。配置文件：[src-tauri/tauri.conf.json](file:///d:/code/mi/dbx/src-tauri/tauri.conf.json)。

### 1.1 标准安装包

生成系统原生安装器（Windows NSIS `.exe` / macOS `.dmg` + `.app` / Linux `.deb` + `.AppImage`）。

**命令：**

```bash
# 完整构建（前端 + Rust + 打包），默认所有目标
pnpm tauri build

# 指定目标架构（跨架构/CI 常用）
pnpm tauri build --target x86_64-pc-windows-msvc
pnpm tauri build --target aarch64-apple-darwin

# 只生成特定 bundle 类型
pnpm tauri build --bundles nsis          # Windows 只生成 NSIS 安装器
pnpm tauri build --bundles dmg            # macOS 只生成 DMG
pnpm tauri build --bundles deb,appimage  # Linux 只生成 deb 和 AppImage
```

底层执行：`pnpm build`（前端打包到 `dist/`）→ `cargo build --release` → `tauri bundle`。

**产物路径与说明：**

| 平台 | 产物路径 | 说明 |
|---|---|---|
| Windows | `src-tauri/target/release/bundle/nsis/DBX_0.5.43_x64-setup.exe` | NSIS 安装器，currentUser 模式，关联 `.sql`/`.db`/`.sqlite` 等文件 |
| Windows | `src-tauri/target/release/dbx.exe` | 裸可执行文件（不打包） |
| macOS | `src-tauri/target/release/bundle/dmg/DBX_0.5.43_aarch64.dmg` | DMG 镜像 |
| macOS | `src-tauri/target/release/bundle/macos/DBX.app` | 应用包 |
| Linux | `src-tauri/target/release/bundle/deb/dbx_0.5.43_amd64.deb` | Debian 包 |
| Linux | `src-tauri/target/release/bundle/appimage/dbx_0.5.43_amd64.AppImage` | AppImage 免安装 |

配置要点（[tauri.conf.json](file:///d:/code/mi/dbx/src-tauri/tauri.conf.json)）：
- `bundle.active: true`，`bundle.targets: "all"` 生成所有格式
- `bundle.createUpdaterArtifacts: true` 同时生成更新签名（`.sig` 文件）
- `windows.nsis.installMode: "currentUser"`（无需管理员权限）
- `windows.nsis.template: "windows/nsis/installer.nsi"` 自定义 NSIS 模板
- 文件关联：`.sql`、`.db`、`.sqlite`、`.sqlite3`、`.duckdb`（L39-53）
- 深度链接协议：`dbx://`（L65-70）

### 1.2 免安装包（Portable）

**仅 Windows 提供**，CI 自动生成。参考 [.github/workflows/release.yml L199-224](file:///d:/code/mi/dbx/.github/workflows/release.yml#L199-L224)。

**本机手动生成：**

```powershell
# 1. 先构建 release 可执行文件（跳过 bundle 阶段）
pnpm tauri build --no-bundle

# 2. 准备 portable 目录
$version = "0.5.43"
$arch = "x64"
$portableDir = "portable/DBX_${version}_${arch}"
New-Item -ItemType Directory -Force -Path $portableDir

# 3. 拷贝产物与标记文件
Copy-Item "src-tauri/target/release/dbx.exe" "$portableDir/DBX.exe"
Copy-Item "LICENSE" "$portableDir/LICENSE"
Copy-Item "README.md" "$portableDir/README.md"
Set-Content -Path "$portableDir/portable.dbx" -Value "" -NoNewline

# 4. 打包为 zip
Compress-Archive -Path "$portableDir/*" -DestinationPath "DBX_${version}_${arch}-portable.zip" -Force
```

**产物：** `DBX_0.5.43_x64-portable.zip`

**内容：**
```
DBX_0.5.43_x64-portable.zip
├── DBX.exe          # 主程序
├── LICENSE          # Apache-2.0 许可证
├── README.md        # 项目说明
└── portable.dbx     # portable 模式标记文件（空文件，存在即启用 portable 模式）
```

**portable 模式行为：** 程序启动时检测同目录下是否存在 `portable.dbx` 文件，存在则将数据目录（配置、连接、传输任务等）放在 exe 同级的 `data/` 目录，而非系统用户目录。适合 U 盘携带、企业内网分发。

**arm64 版本：** 命令中 `$arch = "arm64"`，可执行文件路径为 `src-tauri/target/aarch64-pc-windows-msvc/release/dbx.exe`。

### 1.3 WebView2 离线安装版

适合目标机器无网络或 WebView2 未预装的环境。配置：[src-tauri/tauri.webview2-offline.conf.json](file:///d:/code/mi/dbx/src-tauri/tauri.webview2-offline.conf.json)。

**命令：**

```bash
pnpm tauri bundle --bundles nsis --config src-tauri/tauri.webview2-offline.conf.json
```

**产物：** `src-tauri/target/release/bundle/nsis/DBX_0.5.43_x64-setup.exe`（覆盖默认 NSIS 产物，内嵌 WebView2 引导程序）

**区别：** 标准版依赖系统已安装的 WebView2（Win10+ 通常自带），离线版将 WebView2 引导程序嵌入安装包，体积更大但可在无网络环境安装。CI 会额外上传为 `DBX_{version}_{arch}-webview2-offline-setup.exe`，见 [release.yml L226-255](file:///d:/code/mi/dbx/.github/workflows/release.yml#L226-L255)。

## 2. Web 服务打包

Web 版前后端分离，前端静态文件 + Rust HTTP 服务（[crates/dbx-web](file:///d:/code/mi/dbx/crates/dbx-web)）。

**命令：**

```bash
# 1. 构建前端静态资源到 dist/
pnpm build

# 2. 构建 Web 后端二进制
cargo build --release -p dbx-web
```

**产物：**
- 前端：`dist/`（Vite 产物）
- 后端：`target/release/dbx-web`（Linux/macOS）或 `target/release/dbx-web.exe`（Windows）

**运行：**

```bash
# 设置静态资源目录与密码后启动
$env:DBX_STATIC_DIR="dist"           # Windows
$env:DBX_PASSWORD="your_password"
./target/release/dbx-web.exe

# Linux/macOS
DBX_STATIC_DIR=dist DBX_PASSWORD=your_password ./target/release/dbx-web
```

监听端口默认 `4224`（可通过 `DBX_PORT` 修改）。其他环境变量见 [main.rs L575 附近](file:///d:/code/mi/dbx/crates/dbx-web/src/main.rs#L575)。

**Feature flags：**
- `mq-admin`（默认开启）：启用 MQ 管理功能
- 关闭：`cargo build --release -p dbx-web --no-default-features`

## 3. Docker 镜像打包

Dockerfile 位于 [deploy/Dockerfile](file:///d:/code/mi/dbx/deploy/Dockerfile)，三阶段构建（前端 → Rust 交叉编译 → 运行时）。

**本机构建：**

```bash
# amd64
docker build --platform linux/amd64 -t dbx:0.5.43 -f deploy/Dockerfile .

# arm64（交叉编译，依赖 zigbuild）
docker build --platform linux/arm64 -t dbx:0.5.43 -f deploy/Dockerfile .

# 多架构一起构建并推送
docker buildx build --platform linux/amd64,linux/arm64 -t youruser/dbx:0.5.43 -t youruser/dbx:latest --push -f deploy/Dockerfile .
```

**镜像内容：**
- 基础镜像：`debian:bookworm-slim`
- 运行时依赖：ca-certificates、fontconfig、fonts-dejavu-core、libfreetype6、libssl3
- 二进制：`/usr/local/bin/dbx-web`
- 前端静态资源：`/app/static`
- 数据目录：`/app/data`（环境变量 `DBX_DATA_DIR`）
- 暴露端口：`4224`

**运行：**

```bash
docker run -d \
  -p 4224:4224 \
  -e DBX_PASSWORD=your_password \
  -v dbx_data:/app/data \
  --name dbx dbx:0.5.43
```

CI 流程见 [release.yml L378-499](file:///d:/code/mi/dbx/.github/workflows/release.yml#L378-L499)，构建后推送 Docker Hub 和 CNB Registry 双仓库，多架构 manifest。

## 4. Node 包打包

包含三个子包（见 [package.json](file:///d:/code/mi/dbx/package.json) L12-15）：

| 包 | 路径 | 用途 |
|---|---|---|
| `@dbx-app/node-core` | `packages/node-core` | Node 端核心库 |
| `@dbx-app/cli` | `packages/cli` | 命令行工具 |
| `@dbx-app/mcp-server` | `packages/mcp-server` | MCP Server |

**命令：**

```bash
# 构建所有包
pnpm build:packages

# 测试
pnpm test:packages

# 打包为 .tgz（输出到 /tmp/dbx-pack-check）
pnpm pack:packages

# 发布 dry-run（构建 + 打包，不发布）
pnpm publish:dry-run
```

正式发布由 CI 完成，见 [.github/workflows/mcp-release.yml](file:///d:/code/mi/dbx/.github/workflows/mcp-release.yml) 和 [publish-packages.yml](file:///d:/code/mi/dbx/.github/workflows/publish-packages.yml)。

## 5. CI 自动发布流程

### 触发方式

```bash
# 桌面端发布（推送 vX.Y.Z tag）
pnpm release app    # 交互式，自动 bump + tag + push
pnpm release app patch --yes    # 非交互

# Node 包/MCP 发布
pnpm release packages

# Agents 发布
pnpm release agents
```

详见 [scripts/release.mjs](file:///d:/code/mi/dbx/scripts/release.mjs)。

### 桌面端 release（tag `v*`）

触发 [.github/workflows/release.yml](file:///d:/code/mi/dbx/.github/workflows/release.yml)，矩阵构建：

| 平台 | target | 产物 |
|---|---|---|
| macOS Apple Silicon | `aarch64-apple-darwin` | `.dmg` + `.app` |
| macOS Intel | `x86_64-apple-darwin` | `.dmg` + `.app` |
| Linux x64 | `x86_64-unknown-linux-gnu` | `.deb` + `.AppImage` |
| Linux ARM64 | `aarch64-unknown-linux-gnu` | `.deb` + `.AppImage` |
| Windows x64 | `x86_64-pc-windows-msvc` | NSIS `.exe` + portable zip + WebView2 离线版 |
| Windows ARM64 | `aarch64-pc-windows-msvc` | NSIS `.exe` + portable zip + WebView2 离线版 |

构建后自动生成 GitHub Release Draft，附加产物：
- `DBX_0.5.43_x64-setup.exe` / `DBX_0.5.43_arm64-setup.exe`
- `DBX_0.5.43_x64-portable.zip` / `DBX_0.5.43_arm64-portable.zip`
- `DBX_0.5.43_x64-webview2-offline-setup.exe`
- `DBX_0.5.43_aarch64.dmg` / `DBX_0.5.43_x64.dmg`
- `dbx_0.5.43_amd64.deb` / `dbx_0.5.43_arm64.deb`
- `dbx_0.5.43_amd64.AppImage` / `dbx_0.5.43_arm64.AppImage`
- `latest.json`（updater 自动更新元数据）
- `dbx-jdbc-plugin-{version}.zip`

签名密钥：`TAURI_SIGNING_PRIVATE_KEY_BASE64` 和 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`（GitHub Secrets），用于生成 `.sig` 签名文件供 updater 校验。Release 发布后会清理独立 `.sig` 文件（L257-283）。

## 6. 常见问题

### Q: 首次构建很慢？

Rust 依赖首次编译需要 10-30 分钟（duckdb 编译较慢）。后续增量编译会快很多。CI 使用 [swatinem/rust-cache](https://github.com/swatinem/rust-cache) 缓存 `target/`。

### Q: Windows 构建报 `WebView2Loader.dll` 缺失？

安装 [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)（Win10+ 通常已预装）。或使用 [WebView2 离线安装版](#13-webview2-离线安装版)。

### Q: macOS 构建报签名错误？

本地开发构建可忽略签名，正式发布需配置 Apple 证书（`APPLE_CERTIFICATE`、`APPLE_SIGNING_IDENTITY`、`APPLE_ID` 等 Secrets，见 [release.yml L132-146](file:///d:/code/mi/dbx/.github/workflows/release.yml#L132-L146)）。

### Q: 想跳过 DuckDB 减小体积？

DuckDB 是默认 feature（[src-tauri/Cargo.toml](file:///d:/code/mi/dbx/src-tauri/Cargo.toml) L16）。桌面端关闭：

```bash
pnpm tauri build -- --no-default-features
```

Web 端：

```bash
cargo build --release -p dbx-web --no-default-features
```

注意：关闭后无法使用 DuckDB 相关功能（本地文件查询、跨源 JOIN 等）。

### Q: 如何只构建不打包（生成裸二进制）？

```bash
# 桌面端
pnpm tauri build --no-bundle
# 产物：src-tauri/target/release/dbx.exe

# Web 端
cargo build --release -p dbx-web
# 产物：target/release/dbx-web
```

### Q: portable 模式如何识别？

程序启动时检测 exe 同目录下是否存在 `portable.dbx` 文件（[release.yml L221](file:///d:/code/mi/dbx/.github/workflows/release.yml#L221)）。存在则数据目录使用 exe 同级 `data/`，否则使用系统用户目录（`%APPDATA%/com.dbx.app` 等）。
